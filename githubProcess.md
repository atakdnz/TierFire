# GitHub Process

## Repos

- Use the current repo when the target is clear.
- Check repo roots only when boundaries are unclear, a repo is nested, or the user asks to commit all project changes.
- Commit each repo separately. Public code changes go to the public repo; private docs/planning changes go to the private docs repo.
- If a private docs repo is nested in a public repo, the public repo must ignore that folder.

## Privacy

- Never commit private docs, planning notes, agent instructions, credentials, or scratch files to public repos.
- Private repos may track docs and planning files.

## Commits

- Use present tense: `Add`, `Fix`, `Update`, `Remove`.
- Keep the first line concise.
- Do not mention AI tools or agents.
- Do not use emojis.
- Add a short body only when useful.
- Push after committing unless told not to.
